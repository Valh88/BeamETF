# BeamETF

Full **Erlang External Term Format (ETF)** codec for Object Pascal (FPC/Lazarus).

Supports all standard Erlang/OTP term types, Elixir-specific conventions
(atoms `nil`/`true`/`false`, binary strings, `__struct__` maps), and
RTTI-based automatic mapping between ETF maps and Pascal classes or records
via custom attributes.

## Requirements

- Free Pascal Compiler 3.2+ (tested on 3.3.1)
- Lazarus 2.x (for IDE integration via `.lpk` package)
- FCL units: `Generics.Collections`, `TypInfo`, `Rtti`

## Project Structure

```
src/
  ETF.Types.pas      — base term hierarchy and all ETF tag constants
  ETF.Atom.pas       — global atom cache / interning + per-connection ref table
  ETF.Decoder.pas    — binary ETF decoder  (bytes → TEtfTerm)
  ETF.Encoder.pas    — binary ETF encoder  (TEtfTerm → bytes)
  ETF.Attributes.pas — custom RTTI attributes for field mapping
  ETF.Struct.pas     — registry: Elixir struct name → Pascal class
  ETF.RTTI.pas       — TEtfObjectMapper + generic TEtfMapper<T> (class-based)
  ETF.RecordMap.pas  — TEtfRecordMapper<T> (record-based, stack-friendly)

packages/
  beametf.lpk        — Lazarus package

test/
  testbeam.pas       — FPCUnit test suite (48 tests)
  testrecords.pas    — record type definitions for record-mapper tests
  fpcunitbeam.lpr    — test runner project

examples/
  generate_sample.exs — Elixir script that writes sample.etf
  sample.etf          — binary ETF file generated by the script above
  EtfParser.lpr       — console example: decode, navigate, RTTI-map
```

## Quick Start

### Decode ETF bytes into a term tree

```pascal
uses ETF.Types, ETF.Decoder;

var
  Term: TEtfTerm;
  Bytes: TBytes;
begin
  // Bytes received from Erlang/Elixir over network or file
  Term := TEtfDecoder.Decode(Bytes);
  try
    case Term.Kind of
      etkAtom:    WriteLn(Term.AsAtom.Value);
      etkInteger: WriteLn(Term.AsInteger.Value);
      etkMap:     WriteLn(Term.AsMap.GetByAtom('key').AsString);
    end;
  finally
    Term.Free;
  end;
end;
```

### Encode a term to bytes

```pascal
uses ETF.Types, ETF.Encoder;

var
  Tuple: TEtfTuple;
  Bytes: TBytes;
begin
  Tuple := TEtfTuple.Create;
  Tuple.Add(TEtfAtom.Create('ok'));
  Tuple.Add(TEtfInteger.Create(42));
  Bytes := TEtfEncoder.EncodeToBytes(Tuple);
  Tuple.Free;
end;
```

### Map ETF to a Pascal class (heap, RTTI)

Annotate a `TPersistent` subclass with attributes, add
`{$modeswitch prefixedattributes}`, and use `TEtfMapper<T>`:

```pascal
uses ETF.Types, ETF.Attributes, ETF.RTTI;

type
  [EtfStruct('Elixir.MyApp.User')]
  TUser = class(TPersistent)
  private
    FId: Integer;
    FName: string;
    FActive: Boolean;
  published
    [EtfField('id')]
    [EtfRequired]
    property Id: Integer read FId write FId;

    [EtfField('name')]
    property Name: string read FName write FName;

    [EtfField('active')]
    property Active: Boolean read FActive write FActive;
  end;

var
  User: TUser;
  Bytes: TBytes;
begin
  // Decode ETF bytes into TUser (caller owns result):
  User := specialize TEtfMapper<TUser>.Decode(Bytes);
  try
    WriteLn(User.Id, ' ', User.Name);
  finally
    User.Free;
  end;

  // Encode TUser back to ETF (produces a map with __struct__ key):
  Bytes := specialize TEtfMapper<TUser>.Encode(User);
end;
```

### Map ETF to a Pascal record (stack, zero heap allocation)

For hot paths or embedded data, use `TEtfRecordMapper<T>` with a `record` type.
No `New`/`Free` required — the value lives on the stack.

The record unit must use `{$mode delphi}` and declare
`{$RTTI EXPLICIT FIELDS([vcPublic,vcPublished])}` inside the `type` block:

```pascal
// myrecords.pas  — compiled with {$mode delphi}
uses ETF.Attributes;

type
  {$RTTI EXPLICIT FIELDS([vcPublic,vcPublished])}
  TPointRecord = record
    [EtfField('x')]
    X: Integer;
    [EtfField('y')]
    Y: Double;
    [EtfField('tag')] [EtfAsAtom]
    Tag: string;
    [EtfIgnore]
    Internal: string;
  end;
```

```pascal
// usage in any unit:
uses ETF.RecordMap;

var
  P: TPointRecord;  // on the stack
begin
  // Fill in-place from an already-decoded term:
  specialize TEtfRecordMapper<TPointRecord>.FillFromTerm(P, MapTerm);

  // Or decode bytes directly (returns record by value):
  P := specialize TEtfRecordMapper<TPointRecord>.Decode(EtfBytes);

  // Encode back to bytes:
  Bytes := specialize TEtfRecordMapper<TPointRecord>.Encode(P);

  // Build an Elixir struct map (adds __struct__ key):
  Term := specialize TEtfRecordMapper<TPointRecord>.ToTerm(P, 'Elixir.MyApp.Point');
  Term.Free;
end;
```

### Register Elixir structs

When the decoder sees a `MAP_EXT` with `__struct__`, it automatically produces a
`TEtfElixirStruct`. Register a Pascal class to enable `TEtfMapper` to fill it:

```pascal
uses ETF.Struct;

// Option 1: read EtfStructAttribute from the class automatically
TEtfStructRegistry.Register(TUser);

// Option 2: explicit name
TEtfStructRegistry.RegisterClass('Elixir.MyApp.Product', TProduct);

// Lookup
Cls := TEtfStructRegistry.FindClass('Elixir.MyApp.User');  // returns TUser
```

## Attributes Reference

| Attribute | Target | Effect |
|-----------|--------|--------|
| `[EtfField('key')]` | property / field | Map to this ETF key name (default: lowercase name) |
| `[EtfIgnore]` | property / field | Exclude from encode and decode |
| `[EtfRequired]` | property / field | Raise `EEtfMappingError` if key absent during decode |
| `[EtfStruct('Elixir.Mod')]` | class | Elixir struct name for `TEtfStructRegistry.Register` |
| `[EtfAsAtom]` | string property/field | Encode as `SMALL_ATOM_UTF8_EXT` instead of binary |
| `[EtfAsBinary]` | string property/field | Force `BINARY_EXT` encoding |

Attributes require `{$modeswitch prefixedattributes}` (objfpc mode)
or `{$mode delphi}`.

## Type Mapping

| Pascal type | Decoded from | Encoded as |
|-------------|-------------|------------|
| `Integer`, `Int64`, `Word`, `Byte` | `TEtfInteger` | `SMALL_INTEGER_EXT` / `INTEGER_EXT` / `SMALL_BIG_EXT` |
| `Double`, `Single`, `Extended` | `TEtfFloat` or `TEtfInteger` | `NEW_FLOAT_EXT` |
| `Boolean` | `TEtfAtom` (`true`/`false`) | `SMALL_ATOM_UTF8_EXT` |
| `string` (default) | `TEtfBinary`, `TEtfString`, `TEtfAtom` | `BINARY_EXT` (UTF-8) |
| `string` + `[EtfAsAtom]` | `TEtfAtom` | `SMALL_ATOM_UTF8_EXT` |
| `enum` | `TEtfAtom` (name) or `TEtfInteger` | `SMALL_ATOM_UTF8_EXT` (name) |
| `TObject` descendant | `TEtfMap` | `MAP_EXT` (recursive) |

## Mapper Comparison

| | `TEtfMapper<T>` | `TEtfRecordMapper<T>` |
|---|---|---|
| Target type | `class` (heap) | `record` (stack / inline) |
| RTTI source | `published` properties | public fields + `{$RTTI EXPLICIT ...}` |
| Memory | `Create` / `Free` required | no allocation — fill `var` directly |
| Mode required | `objfpc` or `delphi` | record unit must use `delphi` |
| Nested structs | `TObject` properties | not yet supported |

3. Click **Compile**, then **Use → Add to project**
4. Add the needed units to your `uses` clause: `ETF.Types`, `ETF.Decoder`, `ETF.RTTI`, etc.
